# 如何选择一个值得做的 AI 项目

选择 AI 项目时，不要只看技术热度。一个值得做的项目，至少要满足 4 个条件：

## 1. 痛点真实

用户不应该只是觉得“有点酷”，而是会觉得：

```text
这个东西能帮我省时间、赚钱、学习、找工作或减少麻烦。
```

## 2. 第一版能做小

好的项目可以从很小的 MVP 开始。

不要一开始就做：

```text
完整平台
多租户
团队协作
复杂权限
支付系统
```

先做一个能跑通核心价值的版本。

## 3. 能持续产出内容

适合 GitHub 增长的项目，最好能拆成系列内容：

```text
第 1 篇：为什么做这个项目
第 2 篇：MVP 怎么设计
第 3 篇：核心功能怎么实现
第 4 篇：踩坑记录
第 5 篇：上线复盘
```

## 4. 能形成个人标签

项目不是孤立的。它应该能帮你形成一个标签，比如：

```text
AI 工具作者
AI Agent 实战开发者
AI 求职工具作者
AI 编程效率工具作者
```

如果一个项目不能帮你建立标签，就算做完了，也很难带来长期关注。
